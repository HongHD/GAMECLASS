[관리자 화면]

강의에서 사용할 게임을 개발하고싶어. 크게 두가지로 개발할거야.
관리자 페이지와 사용자페이지야.
우선은 관리자페이지부터 개발을 할거야.
관리자페이지는 여러페이지로 구성이 될거야.
하나하나씩 나와 함께 개발을 해보자.

- 관리자페이지
: 첫번째로 퀴즈 게임의 퀴즈를 등록할 화면이야.
퀴즈그룹(퀴즈를 그룹별로 관리), 퀴즈의 제목, 퀴즈내용 (3000글자 이내), 정답옵션구분 (단답형, 4지선다, OX로 구분) , 정답옵션 (정답옵션구분에 따라 4지선다 선택될 경우 4지선다의 내용을 등록할 수 있도록해줘.), 정답등록, 등록버튼, 문제추가버튼(클릭하면 아래에 위와같이 퀴즈를 등록할 화면이 나옴)

DATABASE에 대해서 알려줄게
MARIA DB를 사용하고있고, 접속주소는 127.0.0.1:3306 이야.
ID, PASSWORD는 root / root 야. 
DB이름은 APP 이고 테이블은 QUIZ야.
QUIZ 테이블의 컬럼 구성은 아래와 같아 .
NO(INT, PRIMARY KEY), GROUP(VARCHAR 50), TITLE(VARCHAR 150), CONTENTS(VARCHAR 3000), OPTION_DISTINC(VARCHAR 10, 옵션구분 항목), OPTION1(VARCHAR 200),  OPTION2(VARCHAR 200),  OPTION3VARCHAR 200), OPTION4(VARCHAR 200), ANSWER(VARCHAR200, 정답등록 컬럼)
위의 OPTION1~4 는 정답옵션구분이 4지 선다로 선택되어졌을 때 등록되어지는 컬럼이야.

웹 화면으로 볼수 있게 개발해줘



등록된 Quiz를 볼 수 있는 List 화면도 개발해줘. List 화면에 Quiz 등록 버튼을 눌러서 Quiz를 등록 할 수 있는 화면도 개발해줘.
디자인을 좀더 고급스럽고 사용자가 사용하기 쉽게 개발해줘. 지금 화면은 사용자가 사용하기에 너무 불편해. List 화면의 데이터를 클릭하면 상세보기화면으로 이동하지말고 List 화면에서 데이터가 보이고 수정도 가능하게 개발해줘




Type 항목은 목록으로 선택할 수 있게 해줘. 그리고 이 화면에서 새로운 Quiz를 등록 할 수 있게 해줘. Contents는 클릭하면 textarea 팝업형태로 떠서 사용자가 입력 할 수 있게 해줘. 그리고 제목표시줄은 가운데 정렬로 해줘
관리자 페이지에서 퀴즈 초기화 버튼을 클릭하면 사용자 화면의 Complete 된 것이 모두 초기화되어서 원래 처음의 화면처럼 나오게 해줘. 필요한 DB 테이블이나 컬럼이 있으면 알려줘. 내가 Table 혹은 컬럼을 추가할거야.

사용자가 로그인을 하면 현재는 /user/main.html 페이지로 접속을 하는데, app.html 페이지로 변경을 해줘. app.html 페이지에는 대기중 이라는 노란색 큰 글씨가 나온다.
관리자가 관리자 페이지에서 '게임 스타트' 버튼을 클릭하면 
그 순간 바로

게임시작 이란 글씨로 바뀌면서 main.html 로 리다이렉트 해줘. 
그리고 전체적으로 사용자페이지는 휴대폰에서 진행하는거야. 그래서 화면을 휴대폰에서 사용할 수 있는 디자인으로 변경해줘


관리자페이지 /admin/quiz_list.html 에 Game Start 버튼 옆에 Game Stop 버튼도 만들어줘. 버튼을 클릭하면 사용자화면이 어떤 페이지에 있던, app.html 화면으로 리다이렉트 되고, 대기중이란 글씨가 표시되게 해줘.

관리자페이지 /admin/quiz_list.html 에 Game Start 버튼 옆에 Game Stop 버튼도 만들어줘. 버튼을 클릭하면 사용자화면이 어떤 페이지에 있던, app.html 화면으로 리다이렉트 되고, 대기중이란 글씨가 표시되게 해줘. 
다시 관리자페이지에서 Game Start 버튼을 클릭하면 사용자는 app.html 로 돌아오기 전 원래 페이지로 이동시켜줘. 


[사용자 화면 ]

사용자 화면을 개발해보려고해. 사용자 접속주소는 /user/index.html 이야.
첫 화면에 접속을 하면 로그인 페이지가 나와. id , password를 입력하고 로그인 버튼을 클릭하면 로그인이 되는 화면이야. 
로그인 db는 동일 DB의 USER 테이블이야.
USER 테이블의 컬럼은 
ID, PASSWORD,TEL(사용자 전화번호), CREATE_DATE (최초 데이터 등록일시), MODIFY_DATE (데이터가 수정된 일시), USE_YN (값은 Y,N)  야.
모두 VARCHAR 50 이고 USE_YN만 CHAR 야. 
사용자가 가입할 회원가입 페이지를 만들어주고, 회원가입되어있는 사용자가 로그인을 하면 ID, PASSWORD가 일치하고 USER\_YN이 Y 인 경우에만 로그인이 되게 해줘. 
로그인이 성공하면 화면은 다시 만들도록 내가 지시할게



user/ 안의 login.html 페이지를 제외한 모든 페이지는 로그인이 되어야만 접속할 수 있게 해줘. 그냥 주소를 입력해서 들어갈 경우에는 로그인 세션이 있는경우에만 정상적으로 접속할 수 있게해주고, 로그인 세션이 없는경우에는 login.html 페이지로 리다이렉트 해줘. 

main.html 의 초기화 화면에 첫번째 그룹만 클릭할 수 있게해줘.
나머지 그룹은 사각형 버튼에 열쇠 모양의 이미지가 레이어 되어져있고 클릭이 되지 않게해줘.
첫번째 그룹에 포함된 문제를 모두 다 풀면 다 푼 그룹 사각형에는 Complet 이란 큰 글자가 레이어 되게 해줘. 그리고 다시 클릭해서 내가 푼 정답을 다시 볼 수 있게 해줘. 첫번째 그룹에 포함된 문제를 모두 다 풀면 두번째 그룹의 문제를 풀수있게 해줘. 세번째는 여전히 열쇠모양의 버튼이 있고, 클릭안되게 해야해. 두번째 그룹의 문제를 다 풀면 세번째 그룹의 문제를 풀수있게 해줘




[기능]

main.html 화면의 모든 그룹의 퀴즈를 다 풀면 화면에 '미션완료' 라는 버튼을 클릭할 수 있게 해줘. 반드시 모든 그룹의 퀴즈를 다 풀어야 활성화 되게해줘. 버튼을 클릭하면 
관리자 페이지에 monitoring.html 파일에 가장 사용자 화면에서 미션완료 버튼을 누른 순서대로 1등 사용자부터 완료된 사용자 순서대로 화면에 표시되도록 화면을 만들어줘. 실시간으로 미션완료 버튼 클릭시 실시간으로 monitoring.html 파일에 표시되어져야만 해. 참여자들이 본인의 등수를 확인할 수 있게 화면을 크게 만들어줘. 
사용자가 여러명인 경우 DB Lock 등의 문제가 발생될 수 도 있어서 성능이 느려질 수 도 있을것같아. 사용자가 여러명이라도 아무런 문제가 발생되지 않게 안정적으로 개발해줘. 매우 안정적으로 성능이 잘 나올 수 있게 만들어줘


admin/monitoring.html 페이지의 왼쪽영역에 사용자가 로그인을 하면 접속자현황 이란 항목을 만들어줘.  사용자 ID만 접속순서대로 보여줘. 필요한 DB 테이블, 컬럼들이 있으면 추가해줘. 
매우 잘했어. 사용자가 로그아웃(logout)을 하면 접속자현황에서 해당 ID를 제거해줘. 그리고 관리자가 접속되어있는 사용자들을 모두 강제 로그아웃 시키는 기능도 만들어줘.





관리자페이지에 접속권한을 추가해줘. 관리자의 로그인 페이지를 별도로 개발해줘. USER테이블과 별개로 관리자 페이지에  접속할 수 있는 별도의 테이블을 만들어줘. 테이블은 'USER_MNG'이고 QUIZ_LIST 테이블에 USER_MNG 의 관리자ID 컬럼도 같이 넣어서 관리자별로 퀴즈문제를 관리 할 수 있게 해줘.



관리자로 로그인을하고 첫 페이지에(신규페이지) 진행시작 버튼을 클릭하면 랜덤숫자(중복 되지 않게) 4자리가 표시된다. 이 랜덤숫자는 관리자가 진행시작 버튼을 클릭하기 전까지는 로그아 웃을 하더라도 유지되어야한다
신규로 만들어지는 첫페이지는 진행시작 버튼(신규로 랜덤숫자 생성)과 기존진행 버튼(무조 건 처음 진행시작 버튼을 통해서 랜덤숫자가 생성된 경우에만 활성화) 이 있다. 
이 두개의 버튼 올 클릭하면 quiz list.html 페이지로 리다이렉트 된다
quiz list.html 화면과 monitoring.html 화면에는 항상 4자리의 랜덤숫자가 좌측 상단에 표 시되어져야한다.
이 랜덤숫자는 다음에 개발할 사용자가 처음 접속할때 랜덤숫자를 그대로 입력해서 해당 관 리자에 속한 사용자라는 것을 알 수 있는 번호이다.


사용자가 로그인을 하면 퀴즈코드 입력창이 뜬다. 관리자가 만들어놓은 코드를 입력하면 코드를 생성한 관리자의 퀴즈만 화면에 보인다.  코드를 생성한 관리자의 퀴즈만 다 풀면 Mission Complete 버튼이 활성화된다. 
관리자페이지에는 본인이 생성한 코드로 입력한 사용자의 정보만 보인다. 강제 로그아웃을 해도 본인이생성한 코드로 입장한 사용자만 로그아웃되고, 게임 Reset Data도 본인이 생성한 코드로 입장한 사용자의 데이터만 Reset 되어야만 한다.

Force logout all 버튼을 클릭하면 해당 관리자가 생성한 코드로 접속한 사용자만 로그아웃이 되게해줘. 지금은 다른 관리자가 생성한 코드로 접속한 사용자까지 모두 로그아웃이되버려. 
그리고 Force logout all 버튼을 클릭해서 사용자가 로그아웃되어도 화면에는 계속 사용자 정보가 남아있고, 새로고침을 해야만 사라져. 실시간으로 사라지게해줘.


관리자가 New Quiz 버튼을 눌러서 새로운 퀴즈를 만들때 Contents에 지금은 텍스트만 입력할 수 있어. 관리자가 Contents에 입력할때 이미지도 등록할 수 있게 해줘. 




왼쪽 빨간색 영역은 게임의 종류를 표시할거야. 사각형 모양 하나하나가 모두 각각 다른 게임 종류야. 
사각형 버튼을 클릭하면 우측에 MISSION COMPLETE RANKING  아래 화면이 게임별로 맞춤혐으로 바뀌게 될거야.
우선은 사각형 하나에 '스피드 게임' 이란 글자를 넣어줘.
스피드 게임 버튼을 클릭하면 MISSION COMPLETE RANKING 영역에 USER들이 버튼을 클릭하면 클릭한 
순서대로 누가 가장 빠르게 눌렀지는가 순위로 표시가 될거야. 관리자가 스피드 게임 버튼을 클릭하면 
USER (관리자가 설정한 CODE 번호를 입력한 사용자에게만) 화면에는 부저버튼이 나오게 해줘야만 해.

admin/monitoring.html 페이지의 Mission Ranking  사각형 버튼을 눌렀는데 왜 자꾸 스피드게임 화면으로 사용자가 나오는거야. 제대로 만들어줘.  Mission Ranking  버튼을클릭한뒤 게임 스타트 버튼을 클릭하면 사용자화면은 user/main.html 페이지로 리다이렉트 해줘.

admin/dashboard.html 페이지의 어떤 버튼을 클릭해도 아무런 반응이 없어. 제대로 수정해줘. 
admin/dashboard.html 페이지의 
Mission Game을 클릭하면 사용자화면은 Mission 을 풀수 있는 화면이 나오고
Speed Game을 클릭하면 사용자화면은 부저를 누를수 있는 화면이 나와야해. 


monitoring_speed.html 화면에서 Reset Game을 클릭하면 사용자화면이 현재는 Mission 을 푸는 화면으로 이동하는데, monitoring_speed.html 이 화면에서의 Reset Game 버튼을 클릭하면 부저를 클릭할 수 있는 화면으로 이동시켜줘.  
즉 Start Game 버튼을 클릭하면 부저를 활성화 시켜주고
Reset Game 버튼을 클릭하면 부저를 비활성화 시켜주는 화면으로 만들어줘.
